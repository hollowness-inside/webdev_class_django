## Реализованный функционал

### Реализованные формы

#### Форма обратной связи (`/feedback/`)
- Поля: имя (до 50 символов), email (валидный формат), тема, текст сообщения (от 10 до 300 символов)
- Валидация: обязательность полей, формат email, минимальная длина текста

#### Форма регистрации (`/register/`)
- Поля: логин, email, пароль, подтверждение пароля
- Валидация: совпадение паролей, формат email

#### Форма входа (`/login/`)
- Поля: логин (≥4 символа), email, пароль
- Валидация: минимальная длина логина, формат email

### Технические особенности

- **CSRF-защита:** Все формы содержат токен `{% csrf_token %}`
- **Валидация:** Реализована как с помощью встроенных средств Django, так и кастомными методами `clean_<fieldname>()`
- **Обработка ошибок:** Ошибки валидации отображаются под соответствующими полями формы
- **Навигация:** Единое меню навигации во всех шаблонах
- **Единый стиль:** Все страницы используют общий базовый шаблон и CSS

## Реализованный функционал

### Модели данных (`fefu_lab/models.py`)

Были созданы модели, полностью соответствующие требованиям задания, с адаптацией названий под тематику проекта:
- **`Member` (Студент):** Хранит информацию о пользователях, включая `first_name`, `last_name`, уникальный `email`, `password` и `clan` (аналог факультета).
- **`Volturian` (Преподаватель):** Модель для преподавателей курсов со полями `first_name`, `last_name`, `ability` и `degree`.
- **`Course` (Курс):** Содержит `title`, `description`, `duration` и `ForeignKey` на модель `Volturian`.
- **`Enrollment` (Запись на курс):** Модель-связка для отношений "многие-ко-многим" между `Member` и `Course`, хранит статус записи.

### Административный интерфейс (`fefu_lab/admin.py`)

Для всех моделей настроена панель Django Admin с расширенной функциональностью:
- **Отображение:** Настроены `list_display` для удобного просмотра ключевых полей.
- **Фильтрация:** Добавлены `list_filter` для фильтрации по статусу, клану, уровню курса и т.д.
- **Поиск:** Реализован `search_fields` для поиска по именам, email и названиям.

### Интеграция с существующими страницами

Все страницы, ранее использовавшие статические данные, были переведены на работу с базой данных через Django ORM:
- **Главная страница:** Отображает динамическую статистику (количество студентов, курсов) и список последних курсов.
- **Страницы студентов и курсов:** Теперь загружают информацию напрямую из базы данных по ID или slug.
- **Удаление констант:** Статические словари `STUDENTS_DATA` и `COURSES_DATA` были удалены, что подтверждает полный переход на ORM.

### Формы и работа с БД

Формы регистрации и входа были обновлены для взаимодействия с БД:
- **Регистрация:** При успешной валидации данные пользователя сохраняются в таблицу `Member`.
- **Валидация по БД:** Реализована проверка на уникальность `email` при регистрации и проверка существования пользователя при входе.

### Управление данными

- **Миграции:** Для всех изменений в моделях созданы и применены миграции.
- **Наполнение БД:** Создана management-команда `seed_data` для автоматического заполнения базы тестовыми данными, что упрощает разработку и тестирование.
